群聊聊天室

需求分析
    有人进入聊天室需要输入姓名，姓名不能重复
    有人进入聊天室时，其他人会收到通知：Lucy 进入了聊天室
    一个人发消息，其他人会收到：   Lucy ： 一起出去玩啊。
    有人退出聊天室，则其他人也会收到通知 :  Lucy 退出了聊天室

    C/S

技术点分析
    * 网络模型　：　udp
    * 服务端存储用户信息 ： name 　address
    　{name:address}
      [(name,address)]
      class User:
        def __init__(self,name,address):
          self.name = name
          self.address = address

    * 消息、通知怎么发送：转发
      父进程发送消息
      子进程接收消息

分功能逻辑设计
　　
　　框架结构串联　　

　　进入聊天室　
       客户端 :  输入姓名
                将姓名发送给服务端表示进入请求进入
                从服务端得到通知是否能进入

       服务端： 接收用户姓名
               判断用户是否重复
               将结果发送给客户端
               OK：通知其他人
                   将用户信息存储

　　聊天
       客户端
           创建子进程
           父进程 循环发送
           子进程 循环接收

       服务端
           接收消息
           转发消息


　　退出聊天室
         客户端： 输入exit或者直接点×

         服务端： 告知其他人
                 将该用户删除
